# Cycle Tracker App Case Study

## Project Summary

This case study documents the design and technical process for building a privacy-first cycle-aware fitness and food companion. The goal is to create a visually rich, technically thoughtful app that helps users understand relationships between cycle phases, food, workouts, energy, symptoms, and recovery.

## Portfolio Goal

Build a polished product that shows prospective employers:

- Strong product thinking
- Modern frontend engineering
- Privacy-aware health data design
- Beautiful visual design
- Data visualization skill
- Clear technical documentation
- A realistic path from web app to mobile app

## Case Study Presentation

Decision docs:

## Current Product Direction

The app should help users answer:

- Where am I in my cycle?
- How is my energy changing?
- How are symptoms, food, workouts, and recovery connected?
- What patterns are showing up over time?
- How can I adjust training and nutrition without making medical claims?

## Build Strategy

Start with a web app first, then prepare for a future mobile version.

Recommended web stack:

- Next.js
- TypeScript
- Tailwind CSS
- shadcn/ui
- Radix UI
- Recharts or Visx
- Framer Motion
- IndexedDB for local-first storage

Recommended future mobile stack:

- Expo
- React Native
- TypeScript
- Expo Router
- SQLite
- SecureStore
