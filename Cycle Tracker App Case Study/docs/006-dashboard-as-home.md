# Dashboard As Home

## Decision

Make the dashboard the first major product screen instead of starting with a marketing-style landing page.

## Why This Matters

The dashboard should immediately show what makes the app useful: the relationship between cycle phase, energy, nutrition, workouts, symptoms, and recovery.

## Dashboard Should Show

- Current cycle day and estimated phase
- Cycle progress
- Energy score
- Recovery score
- Nutrition or food focus
- Water intake
- Sleep trend
- Recent symptoms
- Workout suggestion
- One or two personalized insight cards
- Quick Add action

## Tradeoff

The app needs strong demo data early so the dashboard feels complete before real user input exists.
