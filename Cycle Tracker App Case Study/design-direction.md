# Design Direction

## Design Goal

Create a visually rich app that feels calm, polished, trustworthy, and data-driven.

The app should avoid looking like a generic period tracker. It should feel like a premium wellness dashboard that combines the simplicity of Apple Health, the elegance of Oura, and the warmth of a wellness journal.

## Visual Personality

- Warm
- Intelligent
- Calm
- Premium
- Supportive
- Privacy-conscious

## Color Direction

Primary palette:

- Sage Green: `#7FAF8A`
- Dusty Lavender: `#B9A7D5`
- Background: `#FCFBF8`
- Surface: `#FFFFFF`
- Warm Cream: `#FAF8F4`
- Soft Stone: `#E8E4DE`
- Primary Text: `#2D2D2D`
- Secondary Text: `#727272`

Cycle phase palette:

- Menstrual: Deep Rose `#D97B87`
- Follicular: Soft Green `#A6D7A7`
- Ovulation: Golden Yellow `#F4D77A`
- Luteal: Lavender `#C6B8E8`

## Typography Direction

Use:

- DM Sans for headings
- Inter for body text and UI

Why this matters:

The display font gives the app personality. The body font keeps the app readable.

## Layout Direction

- Use an 8-point spacing system.
- Use 24px screen padding.
- Use 20-24px card padding.
- Use 32px section spacing.
- Use 52px button height.
- Use 20px card radius and 18px button radius.
- Use soft shadows, such as `0 6px 20px rgba(0,0,0,.05)`.
- Treat spacing and dimensions as adaptable design tokens for web and mobile, not fixed rules.

Why this matters:

Consistent spacing and shape make the product feel designed instead of assembled screen by screen. The exact values can adjust by platform so the interface feels natural on desktop, tablet, and phone screens.

## Dashboard Direction

The main app should include:

- Current cycle phase
- Cycle progress
- Nutrition score
- Water intake
- Sleep
- Daily insights
- Quick Add action

Main sections should include:

- Home
- Nutrition
- Cycle
- Insights

Why this matters:

These sections keep the product centered on the relationship between cycle patterns, nourishment, recovery, and daily behavior.

## UI Principles

- Build the product screen first, not a marketing page.
- Make the dashboard feel useful immediately.
- Use cards for repeated information, not every section.
- Keep forms short and friendly.
- Make charts clear on mobile and desktop.
- Use motion to explain changes, not to distract.
- Avoid bouncy animations.
- Use rounded outline icons with a consistent 2px stroke.
- Show empty, loading, and error states.
